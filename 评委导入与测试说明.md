## 三、快速开始

### 3.1 前置条件

- Docker Desktop（或 Docker Engine + Compose 插件）
- Python 3.10+ 及源码依赖：`pip install -r requirements.txt`
- 一个 **OpenAI 兼容的大模型服务 API Key**（默认 provider 及地址见 `configs/config.yaml`；
  LLM 与 Embedding 默认共用此 Key，也可在配置中分别指定）
  （若使用出题方随附的演示 Key，可跳过此项）

### 3.2 一键启动数据库

在项目根目录执行：

```bash
docker compose up -d
```

首次启动时，PostgreSQL 会自动执行 `submission_seed.sql`，完成建表、部署 pgvector
扩展、灌入知识库向量 + 演示账号 + 画像/图谱/路径。等待容器健康检查通过：

```bash
docker compose ps      # STATUS 显示 healthy 即就绪
```

> 数据库连接信息（已与 `.env.example` 对齐，无需改动）：
> `postgres` / `softbei_demo` / 库名 `softbei` / 端口 `5432`。

### 3.3 配置 `.env`

复制 `.env.example` 为 `.env`，其中数据库连接串已可直接使用，仅需填写 Key：

```dotenv
DATABASE_URL=postgresql+asyncpg://postgres:softbei_demo@localhost:5432/softbei
LLM_API_KEY=<大模型服务 API Key（或出题方演示 Key）>
JWT_SECRET=<任意随机字符串>
```

> - `LLM_API_KEY` 同时用于对话生成与检索向量化。
> - 更换 `JWT_SECRET` 不影响演示账号登录（密码为 bcrypt 存储，登录时重新签发 Token）。
> - Embedding 模型由 `configs/config.yaml` 的 `embedding.api_model` 指定，默认输出 1024 维向量。
>   若要更换 Embedding 模型，须保证输出维度仍为 1024，否则查询向量维度与快照中的库向量不一致。

### 3.4 启动并测试

```bash
uvicorn backend.main:app --port 8000
```

浏览器访问 `http://localhost:8000/app` → 用 `demo / demo1234` 登录。

> 首次启动时系统检测到向量库**非空**，不会触发自动索引，直接就绪。

**建议验证路径**：
1. 登录后查看 Dashboard / 学生画像 / 知识图谱 / 学习路径 —— 均已有演示数据；
2. 进入「资源生成」，选择任一知识点（如"卷积运算"）生成学习文档 / 思维导图 / 测验，
   观察 RAG 检索溯源是否引用《动手学深度学习》原文；
3. 做一次测验并提交，查看学习评估与历史记录更新。

### 3.5 停止 / 重置

```bash
docker compose down          # 停止容器，保留数据卷（下次启动数据仍在）
docker compose down -v       # 停止并删除数据卷（下次启动重新从快照导入）
```

## 四、备选方案：源码内启动数据库（不使用 Compose）

若评委已有自己的 PostgreSQL（须为 15+ 且能安装 pgvector 扩展），可手动导入快照：

```bash
# 1. 建库并启用扩展
createdb -h localhost -U postgres softbei
psql -h localhost -U postgres -d softbei -c "CREATE EXTENSION IF NOT EXISTS vector;"
# 2. 导入快照
psql -h localhost -U postgres -d softbei -f submission_seed.sql
# 3. 配好 .env（同 3.3），启动服务
uvicorn backend.main:app --port 8000
```

## 五、本地生成提交快照的步骤（复现用）

以下为制作 `submission_seed.sql` 的一次性流程，各评委无需执行。

```powershell
# 环境：conda activate softbei，Docker 中已有 pgvector 容器
# 1. 新建一个干净的提交库（不污染开发库 softbei）
docker exec <pg容器> psql -U postgres -c "CREATE DATABASE softbei_submission;"
docker exec <pg容器> psql -U postgres -d softbei_submission -c "CREATE EXTENSION IF NOT EXISTS vector;"

# 2. 用 Alembic 在提交库上建表
alembic upgrade head

# 3. 临时把 DATABASE_URL 指向提交库，写入演示账号与图谱/路径，seed_demo.py 会读取环境变量 DATABASE_URL）
python seed_demo.py

# 4. 临时指向提交库启动服务，等待知识库自动索引完成
uvicorn backend.main:app --port 8000

# 5. 导出整库快照为纯 SQL（供 docker-entrypoint-initdb.d 自动执行）
docker exec <pg容器> pg_dump -U postgres -d softbei_submission --no-owner --no-privileges > submission_seed.sql
```

> - 快照只含**数据库结构与数据**，不含 `.env`；打包提交时排除 `.env`、`logs/`、
>   `uploaded_docs/` 等本地产物。
> - 若替换了知识库教材或修改了向量维度，需重跑步骤 4 重新索引后再导出。
